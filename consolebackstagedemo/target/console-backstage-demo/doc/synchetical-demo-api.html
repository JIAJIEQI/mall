综合Demo后台rest接口使用说明

主要包含Manager和Database Control两个模块后台接口的实现（共17个接口），主要对应用户的注册、登录、订单查询以及商品的展示、购买、抢购等业务功能。

一、后台主要接口总览

1、Manager模块

  • signIn – 登录接口
  • signUp – 注册接口
  • userDetail – 用户信息接口
  • goodsList – 商品列表接口
  • goodsDetail – 商品信息接口
  • pay – 支付接口
  • orderList – 订单列表接口

2、Database Control模块

  • querySimpleUserInfoByName – 简单用户信息查询接口（用于登录验证）
  • queryUserDetailInfoById – 用户信息详情查询接口
  • addUser – 增加用户接口
  • queryGoodsList – 商品列表查询接口
  • queryGoodsDetail – 商品信息查询接口
  • pay – 支付接口
  • queryOrdersList – 订单列表查询接口
  • queryBalance – 余额查询接口
  • queryGoodsCount – 商品库存查询接口

二、测试接口详细说明

1、Manager模块

Manager模块错误码说明

    成功：ManagerService.200

    失败：ManagerService.400

1）、登录接口

  • rest接口：v1/rest/signIn
  • 请求类别：post
  • 请求参数：
{

    "userName":"jack",

    "userPwd":"xxxxx"

}
  • 返回结果：
{

    "errCode":"ManagerService.200",

    "resMsg":[

        {

            "userId":5,

            "userName":"Jack"

        }

    ],

    "signInResult":"SignInSuccess"

}
